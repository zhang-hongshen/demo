# 教学包打磨工作流设计

## 目标

把当前“生成结果”提升为老师可以直接备课、调整和带走的教学包。第一版聚焦一个完整闭环：生成结果后，老师可以编辑具体内容、让 AI 只重写当前片段、补充分层任务，并将编辑后的版本导出。

这个功能服务于真实教师工作流，而不是增加更多展示页。教师仍然从现有的“课件生成、作业批改、学情分析”入口开始，生成结果页是本次改造的主要工作区。

## 范围

### 第一版包含

1. 生成结果中的文本字段可编辑，编辑结果保存在浏览器当前页面状态中。
2. 对单个结果片段提供局部重写：更简洁、更适合课堂互动、降低难度、提高难度、重新生成。
3. 课件生成结果增加分层任务：基础巩固、能力提升、挑战拓展。
4. Word、PDF、PPT 导出均使用当前编辑后的结果。
5. Node API 和 Cloudflare Function 同时支持局部重写，并保持现有的密钥保护和错误处理方式。
6. 无法调用模型时，保留用户当前内容，不用错误结果覆盖已有编辑；样例结果也包含分层任务，演示流程仍然可用。

### 明确不包含

1. 登录、账号、云端保存、历史版本和多人协作。
2. 复杂的数组增删、拖拽排序和整套课件的自由排版。
3. 对 PDF、Word、PPT 正文进行解析；现有材料上传能力保持不变。
4. 新增独立的教师工作台、统计后台或学生端。

## 用户体验

### 结果编辑

生成成功后，结果区仍按当前功能展示，但每个可修改的文本片段旁边显示“编辑”入口。进入编辑状态后：

- 单行内容使用输入框，多行内容使用文本域。
- 数组型内容以“一行一项”的方式编辑，保存时转换回原有数组结构。
- 结构化对象中的字段分别编辑，例如课件页的标题和讲稿、测验的题目和答案、评分细则的说明。
- 保存后立即刷新当前功能视图，切换功能和导出都读取同一个编辑后结果对象。
- 不做整页自动保存；刷新浏览器后不保证保留，页面内切换功能不丢失。

### 局部重写

每个可重写片段提供以下动作：

- 更简洁
- 更适合课堂互动
- 降低难度
- 提高难度
- 重新生成

点击后只提交当前片段、必要的课程上下文和动作指令。按钮在请求期间禁用，并在结果区显示处理状态。成功后只替换当前片段，其他内容和教师手工编辑保持不变。每次替换前保留上一个值，提供“撤销本次重写”入口。请求失败时保留原值，并提示“本次重写未应用”。

第一版只允许重写文本叶子字段或一个列表字段，不允许模型直接改写任意 JSON 路径。前端通过受控的目标描述提交字段，后端也校验目标类型和长度，避免局部重写破坏整个结果结构。

### 分层任务

课件生成结果在随堂测验之后增加“课后分层任务”区，包含三组任务：

- 基础巩固：面向尚未掌握核心概念的学生。
- 能力提升：面向已经完成基础目标的学生。
- 挑战拓展：面向需要迁移和探究的学生。

每组任务是可编辑、可重写的任务列表。该区同时出现在课件生成的页面、Word/PDF 导出和 PPT 导出中。

## 数据模型

在现有结果对象上增加可选字段，不改变已有字段的含义：

```js
{
  tieredTasks: {
    basic: ['完成一道核心概念辨析题'],
    advanced: ['用自己的话解释核心方法并完成变式题'],
    challenge: ['将方法迁移到一个真实校园问题']
  }
}
```

生成结果规范化时，缺失的 `tieredTasks` 统一转为空数组。老的模型结果和原始文本结果仍然可以正常显示；没有分层任务时页面显示“暂无内容”，不阻塞其他输出。

局部重写请求使用受控结构：

```js
{
  feature: 'slides',
  target: {
    section: 'slideOutline',
    field: 'speakerNotes',
    index: 0,
    valueType: 'text'
  },
  currentValue: '讲解三种遍历方式。',
  instruction: '更适合课堂互动',
  inputContext: {
    course: '数据结构',
    topic: '二叉树遍历',
    classProfile: '本科二年级，45人'
  }
}
```

接口只返回当前目标的新值：

```js
{ "ok": true, "value": "先让学生预测访问顺序，再用动画验证。" }
```

当 `valueType` 为 `lines` 时，返回值为字符串数组。前端仅在响应校验通过后写入 `currentResult`，并保留目标之外的所有字段。

## 技术设计

### 前端

`public/app.js` 继续作为页面状态协调器，新增三类职责：

1. 将当前结果字段渲染为带有目标描述的编辑控件。
2. 处理编辑保存、撤销和局部重写请求，并对返回值做类型校验。
3. 在课件生成视图和导出前使用同一个 `currentResult`，确保页面、Word/PDF、PPT 不出现不同版本。

编辑器应优先复用现有的 HTML/CSS 结构，增加明确的按钮状态、键盘焦点和错误提示。重写动作采用事件委托，避免为每个结果项重复绑定监听器。

### 服务端

新增共享的局部重写能力：

- `src/promptBuilder.js` 增加 `buildRewriteMessages`，要求模型只返回当前目标的新值，不返回整份教学包。
- `src/deepseekClient.js` 增加局部重写请求构造、调用和响应解析。
- `src/server.js` 增加 `POST /api/rewrite-section`，沿用现有配置、超时和安全错误模式。
- `functions/api/rewrite-section.js` 提供 Cloudflare 对应入口，错误文案不暴露 API 密钥、请求头或模型返回详情。

局部重写 prompt 需要包含课程、主题、班级画像、当前值和动作指令，并要求保留学科准确性。`instruction` 只接受前端定义的五个动作，`target` 只接受已支持的 section/field/valueType 组合；请求体和模型返回值都限制最大长度。

### 导出

`public/exportDocument.js` 和 `public/pptxExport.js` 读取扩展后的结果对象：

- 文档导出增加分层任务区。
- PPT 模型增加一页“课后分层任务”，按三组任务生成要点。
- 已编辑字段无需额外导出参数，因为导出入口已经通过 `currentResult` 传入结果对象。

## 数据流

```text
生成/样例结果
      ↓
  currentResult
      ↓
页面编辑 ───────────────┐
      ↓                  │
局部重写 API → 校验新值 ──┘
      ↓
编辑后的 currentResult
      ├── 页面渲染
      ├── Word/PDF 导出
      └── PPT 导出
```

全量生成失败时沿用现有样例兜底。局部重写失败不触发全量重新生成，也不清空已有内容。

## 错误处理与边界

1. 重写请求缺少目标、动作或当前值时返回 400；模型调用失败返回 502。
2. 前端对返回值做字符串/数组校验；格式不符合目标类型时视为失败。
3. 同一目标同时只能有一个重写请求，避免后返回的旧请求覆盖新编辑。
4. 文本和材料上下文在进入 prompt 前截断到现有安全上限；不把 API 密钥或完整服务端错误传给浏览器。
5. 用户在重写完成前切换功能时，当前请求可以完成，但只有目标仍对应当前结果版本时才应用；否则丢弃结果并提示内容已变化。

## 测试与验收

### 自动化测试

1. `promptBuilder` 测试局部重写 prompt 包含动作、当前值、课程上下文，并要求只返回目标值；课件生成 prompt 包含 `tieredTasks`。
2. `deepseekClient` 测试局部重写请求、文本/数组响应解析和 `tieredTasks` 缺省规范化。
3. `server` 和 Cloudflare Function 测试重写成功、400 参数错误、502 安全错误，确认错误响应不包含密钥。
4. `frontend` 测试编辑后页面状态和导出读取编辑值；重写失败保留原值；过期响应不覆盖新编辑。
5. `exportDocument` 和 `pptxExport` 测试分层任务进入 Word/PDF/PPT 内容。

### 手工验收

1. 使用样例结果编辑一个课件标题、一个测验答案和一个分层任务，切换功能后返回确认内容仍在。
2. 对课件讲稿执行“更适合课堂互动”，确认只改变该讲稿；模拟失败后确认原文保留。
3. 以编辑后的结果分别导出 Word、PDF 和 PPT，确认三种文件包含修改内容及分层任务。
4. 在未配置 API 密钥时确认全量生成和局部重写都显示安全提示，页面仍可加载样例。

## 成功标准

教师不需要重新生成整套方案，就能在一次页面操作中完成“生成 → 修改一处 → 分层布置 → 导出”的流程；所有导出物与页面当前版本一致，模型失败不会损坏已有内容，且现有三项功能和样例兜底保持可用。
